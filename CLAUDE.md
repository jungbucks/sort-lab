# 한눈에 보는 정렬 실험실 — 유지보수 플레이북 (후임 모델용 진입점)

> 수업(빔프로젝터)용 정렬·탐색 시각화. **숫자 카드 3~7장(기본 5, 슬라이더로 조절) + 한 단계씩 실행**이 정체성 — 교사가 설명하며 스텝을 넘기는 도구다.
> 단일 `index.html` · 오프라인 · 외부 요청 0. 사용자 명명: "한눈에 보는 정렬 실험실" (개발 중 별칭 '촵촵').

## 구조 (엔진 핵심만)

- **사전 생성 스텝 방식**: `genSteps(vals, kind, target)`가 알고리즘 전체를 스텝 배열로 미리 시뮬레이션 → `applyStep()`이 하나씩 시각 적용. 새 알고리즘 추가 = genSteps 분기 + (필요 시) 스텝 타입 추가.
- 스텝 타입: compare/swap/fix/key·mark/pick/shift/place(삽입)/range(퀵 구간)/scheck·sdim(탐색)/done·sdone.
- 카드 DOM(2026-07-16 개편, 사용자 지시 "그래프 제거·이동 애니메이션이 주인공") = `.ncard`(균일한 트럼프형 숫자 카드, 높이 150px 고정) 안에 `.crown`(absolute)+`.chk`(✓용 빈 스팬)+값 텍스트. 코너 작은 숫자는 `data-v` 속성 + CSS `content:attr(data-v)` — **textContent를 오염시키지 않아 probe가 값을 그대로 읽는다**(값을 innerHTML 스팬으로 넣지 말 것). 상태색은 `.ncard` 자체의 배경/보더/글자색으로 적용. ❌ 값=높이 막대그래프로 되돌리지 말 것(2026-07-12 명패형도 폐기됨).
- 카드는 슬롯 left 좌표 transition으로 이동(FLIP, .28s = "촵") + 교환 순간 `.hop`(위로 넘음)/`.hop2`(아래로 스침) 340ms 호핑. 상태색: 비교=노랑 · 확정=초록 · 주인공(👑/📍/든 카드)=보라 — 의미 고정, 재배정 금지. 내레이션 숫자는 여백 있는 칩(.narr b).
- 알고리즘 6종: 버블·선택·삽입·퀵(피벗=맨 뒤, Lomuto) + 순차·이진 탐색(**이진은 진입 시 자동 정렬** — "정렬은 탐색을 위한 투자" 서사).

## 규칙

1. 메시지 없는 스텝(msg:'')은 next()가 자동 연쇄 — 정리용 스텝에만 쓸 것.
2. 탐색 모드는 body.search 클래스로 이동 카운터 숨김·목표 배너 표시.
3. 직접 입력은 1~99 서로 다른 **3~7개**(입력 개수에 맞춰 `cardCount` 슬라이더·N 자동 동기화) — 중복·범위 검증 완화 금지(수업 중 오입력 방어). **N은 `let`**(슬라이더가 갱신), 레이아웃·genSteps 전부 N을 호출 시점에 읽으므로 N 변경 후 `resetRun()`만 부르면 재렌더된다.
4. 빔프로젝터 전제: 큰 타이포 유지, 애니메이션은 카드 이동·비교 들썩임까지만.
5. 카드 수에 따라 달라지는 교육 문구는 **`{N}` 플레이스홀더 + `renderExplain()`의 `replaceAll('{N}', N)`** 로 동적 치환(예: 순차탐색 "칸이 {N}개면 최악 {N}번"). N을 하드코딩한 "5" 문구 새로 넣지 말 것.

## 검증

`probe.html`(영구 보존): 6종 자동 완주 + **역순 최악 케이스 검산**(버블 비교10/이동10/최종 1~5 순서). 수정 후 반드시 실행:
`msedge --headless=new --allow-file-access-from-files --virtual-time-budget=60000 --dump-dom probe.html`
